CPU的态和中断机制

发表于 2019-05-20 | 分类于 操作系统

CPU的态

支持操作系统的最基本的硬件结构

  • CPU
  • 内存
  • 中断
  • 时钟

CPU的态(Model)

  • CPU的工作状态

  • 对资源和指令使用权限的描述

    特权指令:

    • LGDT/LIDT: 装载特殊寄存器
    • CLTS: 清除任务开关标志
    • STI/CTI: 允许和禁止中断
    • HALT: 停止CPU的工作
    • IN/OUT: 执行I/O操作
    • 从核态转为用户态

态的分类

  • 核态(Kernel mode)

    • 能够访问所有的资源和执行所有的指令
    • 管理程序内核

  • 用户态(User mode, 目态)

    • 仅能访问部分资源， 其他资源受限
    • 仅能管理用户程序

  • 管态(Supervisor mode)

    介于核态和用户态之间

用户态与内核态之间的转换

  • 用户态向内核态转换

    • 用户请求OS服务
    • 发生中断(在内核态执行)
    • 用户进程错误(内部中断)
    • 用户态企图执行特权命令

  • 核态向用户态转换

    一般是内核态执行完中断指令后返回

CPU读取指令或数据时的访问顺序

  1. 访问缓存(命中, HIT)

  2. 访问内存(没有命中, MISS)

  3. 访问辅存(缺页, PAGE_FAULT)

中断机制

中断定义

  • 是指CPU对突发的外部事件的反应过程或机制
  • CPU收到外部信号(中断信号)后，停止当前工作，转区处理该外部事件，处理完毕后回到原来工作的中断处(断点)继续工作

中断的概念

  • 断点

    • 程序中断的地方，将要执行的下一条指令的地址

    • CS:IP

      程序要执行的下一条指令的地址就是放在这两个寄存器中

  • 上下文

    程序正确运行所依赖的信息集合

    • 相关寄存器

  • 上下文的两个处理过程

    • 上下文的保护: 进入中断服务之前，入栈
    • 上下文的恢复: 退出中断服务程序之后，出栈

中断响应的过程

  1. 识别中断源

    是谁产生了中断

  2. 保护断点和上下文信息

  3. 装入中断服务程序的入口地址(CS:IP)

  4. 进入中断服务程序

  5. 恢复现场和断点

  6. 中断返回: IRET指令

    会把断点弹出到CS:IP寄存器内，只要更新CS:IP的地址，将CS:IP的地址更新为原来断点的地址，那么原来的程序就可以继续运行。

中断响应的实质

  • 交换指令执行地址

    实际上就是CS:IP地址的更新，即当前程序的执行IP与中断程序的执行IP的交换

  • 交换CPU的态

    中断指令是在内核态执行的，所以会发生用户态和内核态的转换
